# 智塔 Strata 竞品与生态调研

调研日期：2026-07-03  
目标产品：智塔 Strata  
产品方向：和用户共创、验证、部署交易策略的智能研究 Agent

## 1. 核心结论

当前市场上已经有不少和“AI + 交易策略 + 回测 + 自动化”相关的产品，但大多数集中在三类方向：

1. 面向散户的自动交易工具：上手容易，但策略解释、研究深度和用户参与感不足。
2. 面向量化工程师的平台：严谨、可回测、可部署，但使用门槛较高。
3. AI 投资助手或信号产品：智能感强，但常偏向“给结论”，不够可验证、可审计、可实施。

智塔 Strata 的机会不在于做一个“AI 荐股工具”，而在于做一个策略研究中间层：

> 用户用自然语言提出想法，Agent 追问并结构化，确定性代码完成回测和信号计算，最后输出可部署的目标仓位和风险解释。

这个定位与普通聊天投顾、拖拽策略工具、专业量化 IDE、黑箱交易信号产品都不同。

## 2. 智塔的建议定位

**智塔 Strata：和用户共创、验证、部署交易策略的智能研究 Agent。**

推荐一句话：

> 你的智能策略塔台：从一句投资想法开始，生成策略、验证历史、输出可部署信号。

产品边界：

- 不直接荐股。
- 不承诺收益。
- 不让大模型直接决定交易。
- 大模型负责共创、追问、结构化和解释。
- 回测、风控、目标仓位、交易信号由确定性代码生成。
- 实盘执行必须经过明确接口和用户确认。

## 3. 产品类型地图

| 类型 | 代表产品 | 用户 | 核心价值 | 主要短板 |
|---|---|---|---|---|
| 自然语言策略自动化 | Composer、Capitalise.ai | 普通投资者、主动交易者 | 用自然语言创建策略并回测/自动化 | 解释和研究深度有限 |
| 技术分析与 AI 信号平台 | TrendSpider、Tickeron | 技术交易者 | 图表、扫描、信号、警报、机器人 | 功能重，容易变成信号消费 |
| 专业量化平台 | QuantConnect、QuantRocket、JoinQuant、BigQuant | 量化研究员、工程师 | 数据、回测、优化、实盘部署 | 门槛高，普通用户难上手 |
| 图表脚本生态 | TradingView | 技术分析用户、策略作者 | 图表、Pine Script、社区和券商连接 | 需要懂指标和脚本 |
| AI 投资研究工作台 | OpenBB、Magnifi、Danelfin | 投研人员、投资者 | 数据查询、AI 分析、评分和研究 | 不专注策略共创和部署 |
| 开源回测/智能体框架 | Backtrader、FinRL-X、LEAN | 开发者、研究者 | 灵活、可扩展、可自建 | 不是终端用户产品 |

## 4. 重点产品分析

### 4.1 Composer

官网：https://www.composer.trade/

定位：AI 投资策略创建、回测和自动交易平台。用户可以用自然语言描述目标、策略和风险，由平台生成策略，并支持自动交易和再平衡。

优点：

- 从想法、策略、回测到执行的闭环完整。
- 无代码体验好，适合非工程用户。
- 策略模板和社区有助于降低启动成本。

缺点：

- 更偏美股/ETF 组合自动化。
- 产品更像“自动投资平台”，而不是研究型 Agent。
- 用户容易直接套用策略，对策略假设、失效条件和风控逻辑参与不足。

对智塔的启发：

- “自然语言 -> 策略 -> 回测 -> 执行”是成立的方向。
- 智塔应强调共创、解释、可审计和风险边界，而不是直接帮用户交易。

### 4.2 Capitalise.ai

官网：https://capitalise.ai/

定位：用日常语言创建、测试、模拟和自动化交易场景。核心是 code-free trading automation。

优点：

- 自然语言条件触发非常贴近用户。
- 适合明确规则，如“如果突破某价格，则执行某动作”。
- 支持测试、模拟、监控和通知。

缺点：

- 更像交易触发器，而不是完整策略研究伙伴。
- 策略质量高度依赖用户表达。
- 对复杂组合、候选策略对比和风险解释不够深。

对智塔的启发：

- 自然语言必须转成结构化、可执行规则。
- 智塔需要通过追问补齐标的、周期、风险、目标、退出规则，而不是直接执行一句话。

### 4.3 TrendSpider

官网：https://trendspider.com/

定位：技术分析、策略测试、AI 策略实验室、AI 助手、警报和交易机器人平台。

优点：

- 技术分析能力强，覆盖图表、扫描、回测、AI 信号和机器人。
- 适合主动交易者验证技术策略。
- AI Strategy Lab 和 Strategy Tester 强化了策略实验能力。

缺点：

- 功能较重，新手容易信息过载。
- 偏技术交易工作站，不是轻量共创 Agent。
- 用户仍需要理解较多指标和交易规则。

对智塔的启发：

- 不应复制“大而全交易终端”。
- 首屏应该只保留用户真正要做的事：想法、约束、策略、验证、部署信号。

### 4.4 QuantConnect

官网：https://www.quantconnect.com/

定位：研究、回测、优化和实盘部署的专业量化平台。其 LEAN 引擎和 Mia AI assistant 代表了专业量化与 AI 辅助的结合。

优点：

- 工程化和专业度强。
- 支持多资产、历史数据、手续费、滑点、保证金和 live trading。
- 适合严肃策略研究和生产部署。

缺点：

- 门槛较高。
- 用户通常需要懂代码、数据和量化框架。
- AI 更像专业助手，不是面向普通用户的产品入口。

对智塔的启发：

- 后端链路要学习专业平台的严谨性。
- 前端入口要避免 IDE 化，要让用户用自然语言开始。

### 4.5 QuantRocket

官网：https://www.quantrocket.com/

定位：Python 量化交易基础设施，支持数据、研究、回测和实盘交易。

优点：

- 数据、回测和执行基础扎实。
- 支持多种回测器和本地/云部署。
- 适合团队自建量化研究系统。

缺点：

- 面向工程用户。
- 没有明显自然语言共创体验。
- 对普通投资者不友好。

对智塔的启发：

- 架构上可参考“数据层、策略层、回测层、执行层”分离。
- 产品上不要暴露太多工程复杂度。

### 4.6 TradingView

官网：https://www.tradingview.com/  
Pine Script 策略文档：https://www.tradingview.com/pine-script-docs/concepts/strategies/

定位：全球市场图表、Pine Script、策略测试器、社区和券商连接平台。

优点：

- 图表体验和社区生态强。
- Pine Script 让技术策略可以被表达、回测和分享。
- 策略测试器能展示收益、回撤、交易列表、买入持有对比等。

缺点：

- 用户仍需要懂指标和脚本。
- 策略共创能力弱。
- 真实交易闭环依赖外部券商和告警。

对智塔的启发：

- 智塔未来可以导出 Pine Script，让用户把策略带到 TradingView 生态。
- 但智塔自己的核心体验应是“自然语言共创 + 本地确定性验证”。

### 4.7 Option Alpha

官网：https://optionalpha.com/

定位：期权策略发现、回测和自动化机器人。

优点：

- 期权场景聚焦。
- “发现机会 -> 回测想法 -> 自动化策略”的路径清晰。
- 教育和模板能力强。

缺点：

- 资产类别较窄。
- 更偏规则执行和机器人管理。
- 不适合直接迁移到 A 股 ETF/个股策略。

对智塔的启发：

- 产品步骤感值得学习。
- 智塔可以先聚焦 A 股/ETF，不必一开始覆盖复杂衍生品。

### 4.8 Tickeron

官网：https://tickeron.com/

定位：AI 交易机器人、AI 虚拟代理、AI 模式识别、趋势预测和交易信号平台。

优点：

- 信号产品化强。
- 用户能快速看到买卖点、胜率、收益等。
- AI 感知明显。

缺点：

- 容易偏黑箱信号消费。
- 用户不一定理解策略为什么成立。
- 如果过度强调收益，容易带来信任和合规风险。

对智塔的启发：

- 智塔不要做神秘的 AI 信号机。
- 每个信号都要能追溯到策略假设、规则、回测和风控。

### 4.9 Danelfin

官网：https://danelfin.com/

定位：AI 股票评分和选股平台，通过 AI Score 评估股票未来跑赢市场的概率。

优点：

- 评分产品简单直接。
- 适合用户快速筛选标的。
- 可解释 AI 是重要卖点。

缺点：

- 更像选股评分，不是策略生成和部署平台。
- 用户参与策略构造较少。
- 与交易规则、仓位管理和回测闭环距离较远。

对智塔的启发：

- 可以借鉴“可解释评分”。
- 但智塔的核心不是评分，而是验证用户自己的策略想法。

### 4.10 OpenBB

官网：https://www.openbb.co/  
GitHub：https://github.com/OpenBB-finance/OpenBB

定位：金融 AI 工作台和金融数据平台，支持数据、研究、API、MCP 和 AI agents。

优点：

- 适合作为金融数据和研究基础设施。
- 对 AI agent 生态友好。
- 可作为智塔后续数据 skill 的候选。

缺点：

- 不是专门的交易策略共创产品。
- 需要用户或团队自行组装工作流。

对智塔的启发：

- 智塔不必自己做所有数据源。
- 可把 OpenBB 这类平台作为外部金融数据 skill。

### 4.11 Backtrader

官网：https://www.backtrader.com/

定位：Python 回测和交易框架。

优点：

- 开源、经典、灵活。
- 适合快速验证策略。
- 可作为智塔未来可插拔回测引擎之一。

缺点：

- 没有面向普通用户的产品界面。
- 没有 AI 共创层。
- 需要写代码。

对智塔的启发：

- 策略执行层可以借鉴其回测范式。
- 但用户不应该直接感知到框架复杂度。

### 4.12 FinRL-X

GitHub：https://github.com/AI4Finance-Foundation/FinRL-Trading

定位：AI-native 模块化量化交易基础设施，覆盖数据处理、策略组合、回测和券商执行。

优点：

- 适合作为智能体交易系统的架构参考。
- “目标权重/目标仓位”接口适合策略和执行解耦。
- 适合强化学习、组合管理和自动化交易研究。

缺点：

- 不是终端用户产品。
- 工程门槛较高。
- 普通用户很难直接使用。

对智塔的启发：

- 智塔部署接口应统一成 target_position 或 target_weight。
- 不要只输出“买入/卖出”，而要输出可执行、可风控的目标仓位。

### 4.13 聚宽 JoinQuant

官网：https://www.joinquant.com/

定位：国内量化投研平台，提供数据、策略研究、历史回测、模拟交易、实盘交易和量化学习体系。

优点：

- A 股数据和中文量化生态强。
- 适合国内用户和国内市场规则。
- 支持研究、回测、模拟和实盘链路。

缺点：

- 偏代码和平台生态。
- 不是自然语言策略共创 Agent。
- 新手仍需要学习量化框架。

对智塔的启发：

- 智塔要优先做好 A 股/ETF、本地中文语境、交易成本、涨跌停、停牌、流动性风险。
- 中文用户需要更自然的策略表达入口。

### 4.14 BigQuant

官网：https://bigquant.com/

定位：AI 人工智能量化投资交易平台，强调数据、因子、机器学习、可视化开发、模拟和实盘。

优点：

- AI 量化定位明确。
- 因子和机器学习能力较强。
- 可视化开发降低了一部分门槛。

缺点：

- 仍偏“大平台”和专业量化。
- 对普通投资者可能过重。
- 自然语言共创不是最核心的体验。

对智塔的启发：

- 智塔可以避开“大而全 AI 量化平台”路线。
- 先做一个更轻、更懂用户想法的策略研究教练。

## 5. Agent Skill 与接口生态

智塔后续可以把能力拆成几类 skill 或接口。

### 5.1 金融数据 Skill

用途：

- 查历史行情。
- 查实时行情。
- 查财务数据。
- 查新闻和公告。
- 查宏观、行业、ETF 成分和资金流。

可参考：

- OpenBB
- Danelfin
- 交易所/行情 API
- 国内数据源，如聚宽、米筐、BigQuant、TuShare 等

### 5.2 回测 Skill

用途：

- 把策略卡转成确定性信号。
- 跑历史收益、回撤、夏普、胜率、交易次数。
- 对比买入持有基准。
- 做参数敏感性和压力测试。

可参考：

- Backtrader
- QuantConnect LEAN
- FinRL-X
- QuantRocket

### 5.3 执行 Skill

用途：

- 将策略信号转为目标仓位。
- 接入 paper trading。
- 接入券商模拟盘或实盘。
- 执行前做风控检查。

可参考：

- Alpaca：https://alpaca.markets/
- Interactive Brokers
- 国内券商交易接口
- 模拟交易环境

### 5.4 策略导出 Skill

用途：

- 导出 Pine Script。
- 导出 Python 策略代码。
- 导出 JSON 策略卡。
- 导出定时任务或 API 配置。

可参考：

- TradingView Pine Script
- QuantConnect LEAN
- Backtrader

### 5.5 研究报告 Skill

用途：

- 生成用户可读报告。
- 解释策略为什么有效。
- 标记策略什么时候可能失效。
- 给下一轮优化建议。

注意：

- 报告不能直接承诺收益。
- 报告要区分历史结果和未来不确定性。
- 报告要明确风险和边界。

## 6. 智塔的差异化机会

### 6.1 从“给答案”转向“共创”

很多产品会直接给策略、信号或评分。智塔应让用户参与策略形成：

- 用户说想法。
- 小塔追问。
- 小塔提炼假设。
- 用户确认约束。
- 系统生成候选策略。
- 历史数据验证。
- 输出下一轮优化建议。

这样产品会更像“智能研究伙伴”，而不是“AI 算命”。

### 6.2 从“黑箱信号”转向“可审计策略”

每个策略至少需要保留：

- 市场假设
- 入场规则
- 退出规则
- 风控规则
- 基准
- 交易成本
- 回测指标
- 失效条件
- 部署信号

### 6.3 从“聊天”转向“可执行接口”

智塔不是只聊天。它应当把自然语言变成结构化对象：

```json
{
  "symbol": "510300",
  "family": "趋势跟踪",
  "horizon": "中期：1-3 个月",
  "risk_profile": "均衡",
  "entry_rule": "20 日均线高于 60 日均线",
  "exit_rule": "20 日均线跌破 60 日均线",
  "risk_rule": "高波动降仓，触发止损后冷却",
  "benchmark": "买入持有",
  "target_position": 0.75
}
```

### 6.4 从“回测结果”转向“下一步行动”

回测后不只显示收益曲线，还要回答：

- 当前最适合哪个方案？
- 为什么它更适合用户目标？
- 它跑赢基准了吗？
- 风险来自哪里？
- 如果要部署，今天目标仓位是多少？
- 如果不部署，下一轮该怎么改？

## 7. 推荐产品路线

### 第一阶段：可信 Demo

目标：让用户相信“智塔能把想法变成可验证策略”。

必须有：

- 自然语言输入
- Agent 追问
- 策略卡
- 多候选策略
- 历史回测
- 买入持有基准
- 风险解释
- 纸面部署信号

### 第二阶段：真实数据与模型共创

目标：让智塔从演示变成可用研究工具。

必须有：

- 大模型 JSON 输出约束
- 实时行情 Provider
- 更多 A 股/ETF 数据
- 参数可调
- 策略导出
- 报告导出

### 第三阶段：模拟交易和执行接口

目标：让智塔具备部署能力，但仍保持风控边界。

必须有：

- paper trading
- 目标仓位接口
- 执行前确认
- 风控检查
- 交易日志
- 策略版本管理

## 8. 对当前项目的直接建议

当前项目已经有：

- 本地 A 股/ETF 数据
- 策略分类
- 策略卡
- 多候选策略
- 回测对比
- 评分
- 研究报告
- 目标仓位信号
- 大模型接口预留

下一步建议：

1. 把产品名统一为 **智塔 Strata**。
2. 把 Agent 口吻统一成“小塔”，减少技术解释。
3. 把策略卡做成可导出的 JSON。
4. 把“部署信号”从展示区升级为接口对象。
5. 增加“为什么不该部署”的风险判断。
6. 增加“下一轮优化”按钮，让用户继续共创。
7. 保留 A 股特色风控：涨跌停、停牌、成交额、滑点、交易成本。
